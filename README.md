## 🛠️ The Core 6-Component Prompt Blueprint (Quick Glance)

[cite_start]According to Chapter 2, an enterprise-grade QA prompt should leverage this exact structural blueprint to ensure deterministic, reproducible results[cite: 477, 492]:

1.  [cite_start]**Role Persona:** Defines the perspective or specific QA character the AI assumes (e.g., Senior Test Automation Engineer)[cite: 478, 479].
2.  [cite_start]**Context:** Background details regarding the system architecture, business requirements, or application tech stack[cite: 480, 481].
3.  [cite_start]**Instruction:** Explicit, clear, and imperative task directives outlining what the model must execute[cite: 482, 483].
4.  [cite_start]**Input Data:** The target baseline data (User Stories, source code, UI wireframes, application screenshots, or logs)[cite: 484, 572].
5.  [cite_start]**Constraints:** Strict negative parameters and boundary conditions detailing what the model *must not* include[cite: 488, 489].
6.  [cite_start]**Output Format:** Defining the precise layout layout scheme (e.g., Gherkin feature files, strict JSON arrays, or Markdown tables)[cite: 490].
